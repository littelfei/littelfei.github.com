js 函数定义方式有三种：
  1. 函数声明语法定义：function sum(num1,num2){return num1+num2}
  2. 函数表达式定义函数：var sum = function(num1,num2){return num1+num2};
  3. new Function构造函数：var sum = new Function(“num1”,”num2”,”return num1+num2”);
    Function构造函数可以接受任意数量的参数，但最后一个参数始终被看成函数体，注意函数表达式定义函数的方法与声明其他变量时一样需要加分号。


各种定义函数方式的区别：

从技术角度上讲，使用Function构造函数定义函数的方式是一个函数表达式，但是不推荐使用这种方式定义函数，原因在于这种方式会导致解析两次代码，影响性能。第一次解析常规的JavaScript代码，第二次解析传入构造函数的字符串。不过，这种语法对于理解“函数是对象，函数名是指针”的概念是非常直观的。
对于函数声明和函数表达式两种定义函数的方式而言，解析器并不是一视同仁的。解析器会率先读取函数声明，并使其在执行任何代码之前可用（可以访问）；对于函数表达式，则必须等到解析器执行到它所在的代码行，才会真正被解释执行。

以上代码的运行结果为20。为什么呢？因为在代码开始执行之前，解析器就已经通过一个名为函数声明提升（function declaration hoisting）的过程，读取并将函数声明添加到执行环境中。对于代码求值时，js引擎在第一遍会声明函数并将它们放到源代码树的顶部。所以，即使声明函数在调用它的代码后，js引擎也能把函数声明提升到顶部。但是，如果把上面的函数声明改成等价的函数表达式，就会在执行期间导致错误。


以上代码会运行错误，因为函数位于一个初始化语句中，而不是一个函数声明。换句话说，在执行到函数所在的语句之前，变量sum中不会保存有对函数的引用；而且，第一行代码运行不通过，实际上也不会执行到下一行。


注意：有名称的函数表达式,如var sum = function add(num1,num2){return num1+num2};也属于函数表达式。
